HTML

  • Serves as like the skeleton of a webpage.
  • The head element contains information about the webpage (aka metadata). Important for SEO (Search Engine Optimization). You can think of it as a brain because you can't see what the body is thinking.
  • The body element represents the visible content shown to the user. Going with the previous brain analogy, these would be like the words that come out your mouth, that everyone can see or hear.
  • The elements can be categorized in a binary fashion of "parental elements" and "child elements, the child elements being within the parental one. And it's all relative, a parent element can alse be see seen as a child element depending on the starting point.
  • The "footer" may logicall seem to be outside of the "body" for non-coders, however it is a child element of the body.
