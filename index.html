HTML

  • Serves as like the skeleton of a webpage.
  • The head element contains information about the webpage (aka metadata).
    • Important for SEO (Search Engine Optimization).
    • You can think of it as a brain because you can't see what the person is thinking.
  • The body element represents the visible content shown to the user.
    • Going with the previous brain analogy, these would be like the words that come out your mouth, that everyone can see or hear.
  • The elements can be categorized in a binary fashion of "parental elements" and "child elements", the child elements being within the parental one.
    • it's all relative, a parent element can alse be see seen as a child element depending on the starting point.
  • The footer element may logically seem to be outside of the body element for non-coders, however it is a child element of the body element.

CSS

  • Serves as fashion stylist to the skleton.
    • This is where the pleasing aesthetics come through.
  • There are 3 different kinds
        CSS incline
        • Uses the style attributess to change the color of a single heading.
        Internal CSS stylesheet
        • Contains CSS rules for the webpage in the head section of the HTML file.
        • Rules apply only to that page
        External CSS stylesheet
        • Allows developpers to keep all their CSS rules in a sperate file
        • Makes design changes easier.
        • Also the one we'll be using for the bootcamp.
  • Below is some useful terminology
    • A declaration has 2 components
      • CSS property we want to apply
      • the value of that property
    • margin
      • how much space we want outside an element
    • padding
      • adds space around the content inside an element
    • *
      • applies a rule to all the elements visible on the page
    • .
      • applies rule to class sector
    • --> keep in mind that if a rule applies to more than one sector, you can code them together as long as you seperate them by a comma.
  • Remember that you must link the CSS file with your HTLM file or else the CSS features will not show up.
    • add a link element inside the head element

Javascript

  • Continuing with the freshly styled skeleton analogy, javasctript is the puppeteer who brings it to life.
    • basically how the users will be able to interact with you, i believe, tbd.
  • A variable is a named container that allows us to store data in our code.
    • Must be unique because it will reference the information stored
    • followed by an equal sign to specify the data.
    • there are various types (here are a few):
      • string
        • sequence of text
        • enclosed in quotation marks
      • number
        • numbers
        • does not need quotation marks
      • Boolean
        • true or false
        • does not need quotation marks
      • array
        • a single variable that is used to hold a group of data
        • always zero-indexed, meaning list starts from 0
  • Control flow is the order in which a computer executes code in a script.
    • read in a linear fashion, from top to bottom
    • can be interrupted by conditional statements and loops
  • Conditional statements start with if (condition){block of code to be executed if condition is truthy}
    • if there's more than one condition, add else if.
  • Loops are used to avoid writing "console.log(var)[0])" for each number (of the things listed in the array)
      • It contains 3 statements
      • 1- starting point of the loop
        • delcare a variable x and give it value 0
      • 2- the condition
        • x < variable.length;
        • as long as the number in the variable x is less than the length of the array, the loop will keep running.
      • 3- the final statement
        • allows the array to iterate over each item
        • we know the number increases by 1 each time
        • code x++ is a shorcut for x+1
          • each code loops, it will increase by 1
      • ends with code block
        • defines the code we want to repeat
          • console.log
  • A function is a set of instructions that tells the computer how to perform a task
    • create a function using the keyword: function
    • followed by its unique name (name used whenever we call the function)
    • followed () (that defines the parameters)
    • follwed by {insert code block}
    • important to note that in order for the code inside of a function to run, it needs to be called
